# Contributing

## Commit Style

Use short, prefixed commit messages so the purpose is clear at a glance:

- `feat:` for new user-facing behavior.
- `fix:` for bug fixes.
- `docs:` for documentation-only changes.
- `chore:` for repository maintenance, release setup, or other non-feature work.
- `refactor:` for code changes that should not alter behavior.
- `test:` for test-only changes.
- `build:` for build system, dependency, or packaging changes.
- `ci:` for GitHub Actions and other automation changes.

Keep the summary in the imperative mood when it reads naturally, for example
`docs: update PC firmware notes`.
